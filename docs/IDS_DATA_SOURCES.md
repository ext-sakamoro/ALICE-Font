# IDS_DATA_SOURCES — IDS データソース調査

ALICE-Font CJK 拡張で利用可能な IDS (Ideographic Description Sequence) データソースの調査と採用方針。

---

## 1. IDS とは

Unicode 標準で定義された漢字構造記述。12 の演算子と部首・漢字を組合せて漢字の構造を表現する。

例:
```
明 = ⿰日月
照 = ⿱⿰日召灬
```

Unicode コードポイント `U+2FF0` - `U+2FFB` (Ideographic Description Characters)。

---

## 2. 主要データソース比較

### 2.1 CHISE IDS

- **URL**: https://www.chise.org/ids/
- **管理者**: 京都大学 大学院情報学研究科 CHISE プロジェクト
- **ライセンス**: GPL v2 (一部 CC-BY-SA)
- **収録字数**: 約 100,000 字 (Unicode CJK 全領域)
- **特徴**: 学術的に厳密、ネスト深い、漢字部位情報も含む
- **問題**: GPL のため MIT/Apache ライセンスの ALICE-Font に直接埋め込み不可

### 2.2 GlyphWiki IDS

- **URL**: https://glyphwiki.org/
- **ライセンス**: パブリックドメイン (CC0)
- **収録字数**: 常用漢字含む数万字
- **特徴**: 日本字形特化、IDS の他に SVG パスも提供
- **取込方法**: API or daily dump
- **採用候補**: ◎ ライセンス互換、データ品質高

### 2.3 Wikimedia / Unicode IRG IDS

- **URL**: https://www.unicode.org/charts/PDF/U2F800.pdf (補助)
- **ライセンス**: Unicode License (MIT 互換)
- **収録字数**: Unicode CJK Compatibility 含む
- **特徴**: 標準準拠、最小限の情報
- **採用候補**: ○ ライセンス互換、ただし IDS 注釈は不完全

### 2.4 ids.txt (CJK Decomposition Database)

- **URL**: https://github.com/cjkvi/cjkvi-ids
- **ライセンス**: Apache 2.0
- **収録字数**: 約 90,000 字
- **特徴**: 簡潔な TSV フォーマット、Unihan データベース由来
- **採用候補**: ◎ ライセンス互換、扱いやすい

## 3. 採用方針

### 3.1 1次ソース: cjkvi-ids (Apache 2.0)

```
ext-sakamoro/ALICE-Font/data/
└── cjkvi-ids/
    ├── ids.txt          # 取込元 (Apache 2.0 by CJKVI)
    └── LICENSE-cjkvi    # ライセンス全文
```

`ids.txt` のフォーマット (TSV):
```
U+4E00  一  一
U+660E  明  ⿰日月
U+68EE  森  ⿱木林
...
```

### 3.2 2次ソース (補完): GlyphWiki (CC0)

cjkvi-ids で IDS 不明な字、または日本字形と異なる場合に GlyphWiki を参照。

### 3.3 取込スクリプト

```rust
// tools/import_ids/src/main.rs
use std::fs::File;
use std::io::{BufRead, BufReader, Write};

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let ids_txt = File::open("data/cjkvi-ids/ids.txt")?;
    let joyo_list = File::open("data/joyo_kanji.txt")?;  // 常用漢字 2,136 字リスト
    let mut output = File::create("../../src/cjk/ids_db_generated.rs")?;

    let joyo: std::collections::HashSet<char> = BufReader::new(joyo_list)
        .lines()
        .flatten()
        .filter_map(|line| line.chars().next())
        .collect();

    writeln!(output, "// Auto-generated by tools/import_ids")?;
    writeln!(output, "// Source: cjkvi-ids (Apache 2.0)")?;
    writeln!(output, "use super::KanjiDef;")?;
    writeln!(output, "pub const JOYO_KANJI: &[KanjiDef] = &[")?;

    for line in BufReader::new(ids_txt).lines().flatten() {
        let parts: Vec<&str> = line.split('\t').collect();
        if parts.len() < 3 { continue; }

        let ch = parts[1].chars().next().unwrap();
        if !joyo.contains(&ch) { continue; }

        let ids = parts[2];
        writeln!(
            output,
            r#"    KanjiDef {{ codepoint: '{}', ids: "{}", stroke_count: 0, joyo_grade: None }},"#,
            ch, ids
        )?;
    }

    writeln!(output, "];")?;
    Ok(())
}
```

## 6. データの版管理

### 6.1 取込タイミング

- S5 開始時に 1 回 (cjkvi-ids の master HEAD を `git submodule` or vendoring)
- 上流の重大な修正があれば更新
- 自動更新はしない (テスト回帰の懸念のため)

### 6.2 バージョン記録

```rust
// src/cjk/ids_db_generated.rs (auto-generated)
pub const IDS_DATA_VERSION: &str = "cjkvi-ids@a1b2c3d (2026-05-14)";
```

### 6.3 vendoring 戦略

ALICE-Font リポジトリに `data/cjkvi-ids/` として **コピーを保持** (submodule ではなく)。
理由: ビルド時にネットワーク不要、CI 安定、ライセンス遵守 (LICENSE 含む)。

サイズ: cjkvi-ids/ids.txt は ~5 MB → リポジトリに含めて OK。

## 7. データ品質チェック

### 7.1 IDS パース検証

```rust
#[test]
fn all_imported_ids_parse_successfully() {
    for def in JOYO_KANJI {
        let result = parse(def.ids);
        assert!(
            result.is_ok(),
            "IDS parse failed for {} (codepoint {:?}): {}",
            def.codepoint, def.codepoint as u32, def.ids
        );
    }
}
```

### 7.2 完全性チェック

```rust
#[test]
fn all_joyo_kanji_covered() {
    let joyo: std::collections::HashSet<char> = load_joyo_list();
    let covered: std::collections::HashSet<char> =
        JOYO_KANJI.iter().map(|d| d.codepoint).collect();
    let missing: Vec<_> = joyo.difference(&covered).collect();
    assert!(
        missing.is_empty(),
        "Missing IDS for {} joyo kanji: {:?}",
        missing.len(), missing
    );
}
```

### 7.3 部首参照の整合性

```rust
#[test]
fn all_ids_components_are_known() {
    let known_components: std::collections::HashSet<char> =
        radicals::RADICALS.iter().map(|r| r.character).collect()
        .union(&JOYO_KANJI.iter().map(|d| d.codepoint).collect())
        .copied().collect();

    for def in JOYO_KANJI {
        let tree = parse(def.ids).unwrap();
        for leaf in tree.leaves() {
            assert!(
                known_components.contains(&leaf),
                "Unknown component {} in IDS for {}",
                leaf, def.codepoint
            );
        }
    }
}
```

## 8. 代替戦略 (バックアッププラン)

cjkvi-ids が利用不可になった場合の代替:

1. **GlyphWiki dump** を 1 次ソースに切替 (CC0、自由)
2. **手書き IDS** で必要分を順次定義 (extoria.co.jp で使う字に限定)
3. **Unihan データベース** から派生情報を取込 (Unicode License)

いずれの場合も ALICE-Font の API は変更不要 (データソース置換のみ)。

## 9. 取込スクリプトの実行手順 (S5+S6)

```bash
# 1. データソース取得 (S5 開始時)
cd ~/ALICE-Font
mkdir -p data/cjkvi-ids
curl -L https://raw.githubusercontent.com/cjkvi/cjkvi-ids/master/ids.txt \
    -o data/cjkvi-ids/ids.txt
curl -L https://raw.githubusercontent.com/cjkvi/cjkvi-ids/master/LICENSE \
    -o LICENSES/cjkvi-ids.LICENSE

# 2. 常用漢字リスト (文化庁 PDF から事前抽出済み)
# data/joyo_kanji.txt として手動配置

# 3. 取込スクリプト実行
cd tools/import_ids
cargo run --release -- \
    --ids-input ../../data/cjkvi-ids/ids.txt \
    --joyo-list ../../data/joyo_kanji.txt \
    --output ../../src/cjk/ids_db_generated.rs

# 4. 整合性チェック
cd ../..
cargo test --features std cjk::ids_db
```

出力された `src/cjk/ids_db_generated.rs` は `git` 管理し、生成スクリプトの再実行は人手判断で行う。
